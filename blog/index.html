JS 的任务管理机制

November 6, 2021 · One min read
Zeffon Wu
A web engineer

JavaScript 是被设计为单线程的，这意味着任务只能一个一个处理。如果某一个任务是 I/O 操作类型的，比如用户输入，那么该操作未完成前，其他所有任务都处于等待状态，这样就会造成页面假死状态，从而导致用户体验极差。为了解决该问题， JavaScript 将任务分为同步任务和异步任务，确保页面的交互和渲染处理能高效进行。
